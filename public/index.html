ACCESS-CI MCP Servers - Data Usage Guide

The ACCESS-CI MCP (Model Context Protocol) servers provide structured access to valuable research computing data. This guide explains how this data can be consumed and integrated into various applications and workflows.

Data Available

Affinity Groups Data

Each affinity group in ACCESS-CI represents a computing resource or community. The data includes:

  • Basic Information: Name, description, coordinator contacts, tags
  • Events & Training: Upcoming workshops, webinars, training sessions with dates, registration links
  • Knowledge Base: Documentation, guides, best practices for each resource

Use Cases & Applications

Research Computing Resource Discovery

Problem: Researchers need to find appropriate computing resources for their work.
Solution: Query affinity groups to discover:
  • Available HPC systems (Bridges-2, Expanse, etc.)
  • System capabilities and specializations
  • Contact information for support
Example Application: 
// Find GPU-enabled systems for ML research
const groups = await getAllAffinityGroups();
const gpuSystems = groups.filter(g => 
  g.tags.includes('gpu') || g.description.includes('GPU')
);

Training & Event Aggregation

Problem: Scattered training opportunities across different ACCESS resources.
Solution: Aggregate all events into a unified calendar:
  • Collect events from all affinity groups
  • Filter by skill level, topic, or date
  • Create personalized training paths
Example Integration:
  • Google Calendar feed of all ACCESS training
  • Slack bot announcing relevant workshops
  • Email digest of upcoming events by topic

Knowledge Base Search Engine

Problem: Documentation spread across multiple resources.
Solution: Build a unified search interface:
  • Index all knowledge base articles
  • Tag by resource, topic, and difficulty
  • Provide contextual recommendations
Example Features:
  • "Find all MPI documentation across ACCESS resources"
  • "Show beginner guides for GPU programming"
  • "Compare storage options across different systems"

AI-Powered Research Assistant

Problem: Researchers need quick answers about ACCESS resources.
Solution: Use MCP with Claude or other LLMs to:
  • Answer questions about specific resources
  • Recommend appropriate systems for workloads
  • Guide through allocation requests
  • Explain technical concepts from KB articles
Example Queries:
  • "Which ACCESS resource is best for genomics workflows?"
  • "What training is available for quantum computing?"
  • "How do I transfer data between Bridges-2 and Expanse?"

Integration Patterns

MCP Integration with Claude

For AI-powered applications, add to your Claude Desktop configuration:

{
  "mcpServers": {
    "access-ci": {
      "command": "npx",
      "args": ["@access-mcp/affinity-groups"]
    }
  }
}

Direct API Integration

For applications that can't use MCP, access the underlying APIs directly:

import requests

# Get affinity group info
response = requests.get(
    "https://support.access-ci.org/api/1.0/affinity_groups/bridges2.psc.access-ci.org"
)
group_data = response.json()

# Get upcoming events
events_response = requests.get(
    "https://support.access-ci.org/api/1.1/events/ag/bridges2.psc.access-ci.org"
)
events = events_response.json()

Available MCP Tools

get_affinity_group Tool

Get basic information about a specific ACCESS-CI affinity group

Parameters:
  • group_id (string, required) - The affinity group ID (e.g., "bridges2.psc.access-ci.org")

get_affinity_group_events Tool

Get events and trainings for a specific ACCESS-CI affinity group

Parameters:
  • group_id (string, required) - The affinity group ID

get_affinity_group_kb Tool

Get knowledge base resources for a specific ACCESS-CI affinity group

Parameters:
  • group_id (string, required) - The affinity group ID

Best Practices

Caching Strategy

  • Cache affinity group data for 24 hours
  • Cache events for 1 hour (more dynamic)
  • Cache KB articles for 1 week

Rate Limiting

  • Respect API rate limits
  • Implement exponential backoff
  • Use bulk queries where possible
